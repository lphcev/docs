useState ​

异步处理和闭包快照 ​

虽然在 handleClick 设置了 setCount(0), 但是每一次点击时, 状态会正常更新 1, 2, 3...

ts
const App = () => {
  const [count, setCount] = useState(0)

  const handleClick = () => {
    setCount(0)
    setCount(count + 1)
    setCount(count + 1)
    setCount(count + 1)
    console.log(count)
  }

  return (
    <>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </>
  )
}

结论: setCount(0) 确实执行了，但瞬间被后面的 setCount(count + 1) 覆盖掉了. 而且因为 count 在函数内部是 "冻结" 的固定值, 所以每次点击最终都会让 count 增加 1

下面拆解点击时的完整执行流程

  1. 批处理导致合并覆盖

React 为了性能, 会把同一个事件函数中的多个 setState 合并成一次更新. 当你点击按钮时, React 并不是每调用一次 setCount 就立刻去改界面上的数字, 而是把你这 4 个操作打包在一起处理. 打包时, 后面的更新会覆盖前面的更新 (对于相同的 state 来说)

  1. 闭包快照: count 是一个常量

handleClick 函数执行期间, count 的值是当前渲染周期的快照, 它不会变

假设当前页面显示的是 Count: 0, 那么在整个 handleClick 运行过程中, count 始终等于 0

代码计算过程 (第一次点击)

  • 执行 setCount(0), 将 count 设为 0

  • 第一次执行 setCount(count + 1), 原先的 count 是 0, 计算结果为 0 + 1 = 1, 将 count 设为 1

  • 之后两次执行, count 值始终都是旧值 0, 执行之后, 结果都为 1

  • console.log(count) 打印的 count 是旧值 0

React 最终取最后一次操作的结果 1 去更新界面. 所以页面从 0 变成了 1

  1. 为什么每次点击都会 "自增"?

因为下一次点击时, 页面上的 count 已经变成了 1

这时再次点击, 新的 handleClick 中的 count 快照值就变成了 1. 代入计算 (第二次点击)

  • setCount(0): 虽然将 count 设为 0, 但是之前的旧值仍然是 1
  • setCount(count + 1) -> 计算为 2，覆盖掉 0, 最终页面变成 2
